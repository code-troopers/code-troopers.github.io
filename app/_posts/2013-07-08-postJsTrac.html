jsTrac est disponible depuis quelques mois sur le dépôt github de SRMvision. jsTrac est un outil pour faciliter la recette de sites web en créant rapidement des rapports de bug sur le logiciel de gestion des incidents trac. Les principales fonctionnalités sont les suivantes :

  • impression d'écran de la page visionnée ;
  • mise en évidence de zones à l'aide de rectangles colorés et de flèches ;
  • ajout d'annotations ;
  • masquage d'informations ;
  • pré-remplissage du formulaire de création d'incidents avec des informations utiles (url par exemple) ;
  • mise-à-jour des incidents.

Lorsque nous travaillons sur un site web, nous faisons beaucoup de recette. Nous avons besoin du login de l'utilisateur, de l'adresse de la page et d'une impression d'écran pour aider à la résolution du bug. Pour nous simplifier ce travail, nous avons initié un stage afin de créer jsTrac. Depuis, nous essayons de le faire évoluer dès que nous le pouvons.

Présentation de l'application

Lancement des scripts

Dans SRMvision, nous donnons accès à jsTrac pour les développeurs uniquement. Pour lancer les scripts, nous utilisons un bouton ou un raccourci clavier. Dans le fichier d'exemple du projet, il s'agit d'un formulaire classique. On pourrait aussi l'imaginer sous forme d'un bookmarklet ou d'un script grease-monkey, donc complètement décorellé du site visité.

Modification de l'image à prendre

Lorsque nous lançons les scripts, une boîte à outils s'ouvre nous offrant la possibilité de travailler la page.

Finitions

Une fois nos modifications validées, une image est générée, les informations de trac sont pré-remplies, l'image est liée… il n'y a plus qu'à remplir un descriptif si nécessaire.

Notre ticket est créé dans trac avec l'image liée.
